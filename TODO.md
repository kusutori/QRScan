# QRScan 功能规划与开发任务清单

这是一个关于将 `QRScan` 从简单的“链接转二维码”工具扩展为全功能跨平台（桌面+移动）二维码工具的规划草案。

## 1. 响应式布局与 UI 重构 (当前重点)
目标：从单一的“居中卡片”布局转变为适应不同屏幕尺寸的现代应用布局。

- [ ] **App 整体架构调整**
    - [ ] 引入路由系统 (如果需要多页面) 或 状态驱动的视图切换。
    - [ ] **桌面端 (Desktop):** 采用 `侧边栏 (Sidebar)` + `内容区` 的布局。左侧导航，右侧为功能区。
    - [ ] **移动端 (Mobile):** 采用 `底部导航栏 (Bottom Tab Bar)` 或 `汉堡菜单`。
- [ ] **生成页面的响应式改造**
    - [ ] **宽屏模式:** 左侧输入表单，右侧实时预览二维码（分栏布局）。
    - [ ] **竖屏/手机模式:** 上下布局，输入在上方，生成的二维码在下方或弹窗展示。
- [ ] **暗黑模式 (Dark Mode)**
    - [ ] 利用 Tailwind 的 `dark:` 类完善深色主题适配。
    - [ ] 添加手动切换主题的开关。

## 2. 核心功能增强：扫码能力
目前应用叫 "QRScan" 但只能生成。我们需要添加扫描功能。

- [ ] **摄像头扫描**
    - [ ] 集成 Tauri 插件或 Web API 调用摄像头读取二维码。
    - [ ] **移动端:** 调用手机后置摄像头。
    - [ ] **桌面端:** 调用 Webcam 或允许“截屏扫描”。
- [ ] **图片识别**
    - [ ] 支持从相册/文件管理器选择图片进行识别。
    - [ ] **桌面端:** 支持直接拖拽 (Drag & Drop) 图片到窗口进行识别。

## 3. 生成功能扩展 (不仅限于 URL)
支持更多类型的二维码生成。

- [ ] **Wi-Fi 连接码:** 输入 SSID 和密码，扫码直接连 WiFi。
- [ ] **电子名片 (vCard):** 生成包含姓名、电话、邮箱的二维码。
- [ ] **纯文本:** 简单的文本信息。
- [ ] **邮件/短信:** 预设接收人和内容的二维码。

## 4. 二维码美化 (个性化)
让二维码不再只是黑白的。

- [ ] **颜色自定义:** 选择前景色（不再只是黑色）和背景色。
- [ ] **嵌入 Logo:** 允许用户上传图片放在二维码中间。
- [ ] **样式调整:** 码点形状（方点、圆点）、容错率设置。

## 5. 历史记录与数据管理
- [ ] **历史记录优化:** 区分“生成记录”和“扫描记录”。
- [ ] **导出功能:** 批量导出历史记录。
- [ ] **收藏夹:** 允许标记常用的二维码。

## 6. Tauri 原生能力集成
- [ ] **分享功能:** (移动端) 调用系统原生分享菜单分享二维码图片。
- [ ] **快捷键:** (桌面端) Ctrl+V 直接识别剪贴板中的链接或图片。

### 讨论建议
我建议我们可以按以下顺序进行：
1. 先进行 **UI 布局重构**，搭建好桌面和手机通用的骨架。
2. 随后实现 **摄像头/图片扫描** 功能，名副其实。
3. 最后添加 **WiFi/名片** 等高级生成功能。
